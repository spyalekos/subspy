# Οδηγίες & Κανόνες Ανάπτυξης AI (AI Reminders)

*Αυτό το αρχείο διαβάζεται και λαμβάνεται υπόψη από τον AI agent κατά την εκκίνηση του τρέχοντος project για την αποφυγή επαναλαμβανόμενων διευκρινίσεων.*

## Βασικοί Κανόνες (Σημαντικοί)

1. **Package / Dependency Management:**
   - Χρησιμοποιούμε **ΠΑΝΤΟΤΕ** `uv`.
   - Δεν τρέχουμε «γυμνές» pip εντολές. Όταν θέλουμε να προσθέσουμε κάτι ή να τρέξουμε κάτι, αξιοποιούμε εντολές όπως `uv pip install ...` ή `uv add ...` ή `uv run ...` αναλόγως του αν διαχειριζόμαστε project file (`pyproject.toml`) ή τοπικό venv.
2. **Framework:**
   - Το project χρησιμοποιεί τη βιβλιοθήκη **Flet** (Material design UI). Πρέπει να παραμένουμε σε προδιαγραφές που λειτουργούν ομαλά με τις νεότερες εκδόσεις του Flet.

3. **Pyinstaller & Build:**
   - Δημιουργούμε εκτελέσιμα (.exe) χρησιμοποιώντας **αποκλειστικά** την εντολή: `uv run pyinstaller <όνομα_αρχείου.spec>`.
   - **Η τρέχουσα (τελευταία) έκδοση** του spec αρχείου είναι: `main.spec`. Πάντα χρησιμοποιούμε αυτή εκτός αν ζητηθεί παρέμβαση προσωποποιημένη.
4. **Εκδόσεις (Versioning):**
   - Οι εκδόσεις του προγράμματος (version) αυξάνονται πάντοτε κατά **0.01** και φροντίζουμε να παράγουμε/αποθηκεύουμε releases κάθε φορά.
   - **ΚΡΙΣΙΜΟ:** Σε **ΚΑΘΕ αλλαγή κώδικα**, η έκδοση αυξάνεται κατά 0.01 και ενημερώνονται **ΟΛΕΣ** οι αναφορές:
     - `pyproject.toml` → `version`
     - `src/version.py` → `VERSION` (Η έκδοση πρέπει να είναι **πάντα hardcoded** σε αυτό το αρχείο, π.χ. `VERSION = "1.0.1"`, και να ΜΗΝ διαβάζεται δυναμικά από το `pyproject.toml`, για να λειτουργεί σωστά το εκτελέσιμο που παράγει ο PyInstaller).
     - Δημιουργία νέου `main.spec` (εφόσον απαιτείται)
     - `main.spec` → το πεδίο `name` παραμένει **σταθερό** (`'subspy'`) χωρίς την αναγραφή της έκδοσης.
     - `AI-Instructions.md` → αναφορά spec αρχείου
     - `README.md` (ή άλλα έγγραφα τεκμηρίωσης)
     - Οποιοδήποτε άλλο αρχείο αναφέρει version.
5. **Στοιχεία Συγγραφέα (Author):**
   - Για το πεδίο του συγγραφέα (Author) στο About Box, χρησιμοποιούμε **ΠΑΝΤΟΤΕ** τα στοιχεία που ανακτούμε από το GitHub μέσω της εντολής `gh api user`.
6. **UI Icons (Flet):**
   - Αντί για `flet.controls.material.icons` πρέπει **ΠΑΝΤΟΤΕ** να χρησιμοποιούμε τα **`ft.CupertinoIcons`**, τα οποία είναι διαθέσιμα με ασφάλεια σε κάθε build εκδοχής.
7. **Git / GitHub:**
   - Εάν ζητηθεί να γίνει ενημέρωση στο GitHub repository, χρησιμοποιούμε το Command Line Tool **`gh`** που είναι ήδη εγκατεστημένο και συνδεδεμένο στο σύστημα.
8. **Πολυγλωσσία (Bilingual Project):**
   - Το Project είναι δίγλωσσο (Ελληνικά και Αγγλικά).
   - **ΠΑΝΤΟΤΕ** κάνουμε ενημερώσεις και στις 2 γλώσσες (αρχεία μεταφράσεων π.χ. `src/i18n.py` και README) με κάθε νέα λειτουργικότητα ή αλλαγή που απαιτεί κείμενο.

## Πιθανά Προβλήματα & Σημεία Προσοχής (Gotchas)

1. **Διαχείριση Αρχείων (PyInstaller):**
   - Για να λειτουργεί η εφαρμογή όταν γίνεται μεταγλώττιση σε `exe`, η φόρτωση τοπικών αρχείων (π.χ. ρυθμίσεις) πρέπει να λαμβάνει υπόψη το `sys._MEIPASS` (αν είναι bundled resources).
   - *Προσοχή:* Αν γίνει αλλαγή της δομής φακέλων, πρέπει να ανανεωθούν και τα datas στο `main.spec`.

---
2. **Διαχείριση Clipboard (Flet):**
    - Η χρήση του `ft.Clipboard()` ως control στο overlay ενδέχεται να προκαλέσει σφάλμα "Unknown control" σε ορισμένες εκδόσεις του Flet client.
    - **Κανόνας:** Χρησιμοποιούμε την ενσωματωμένη υπηρεσία `page.clipboard.set(text)` για μέγιστη συμβατότητα.
    - Η κλήση πρέπει να γίνεται πάντα ασύγχρονα (`await`) μέσα σε `async` handlers.
    - **Flet UI:** Για το άνοιγμα συνδέσμων, χρησιμοποιούμε `await page.launch_url(url)` μέσα σε `async` handlers.
    - **Flet UI:** Όταν ένα control χρησιμοποιεί `content` (π.χ. `ElevatedButton` με `Row`), οι ενημερώσεις κειμένου πρέπει να γίνονται στο εσωτερικό `ft.Text` control και όχι στην ιδιότητα `text` του γονέα.
    - **Flet UI:** Χρησιμοποιούμε ΠΑΝΤΟΤΕ `ft.Colors` (με κεφαλαίο C) για τα χρώματα αντί για `ft.colors`.
    - **Flet UI:** Αποφεύγουμε τις ιδιότητες `cursor` ή `mouse_cursor` σε `Container` controls αν προκαλούν TypeError, καθώς η υποστήριξή τους διαφέρει ανά έκδοση.
    - **Flet UI:** Το `ProgressRing` χρησιμοποιεί `width` και `height` αντί για `size`.
    - **Flet UI:** Αν οι σταθερές του `ft.alignment` λείπουν, χρησιμοποιούμε τον constructor `ft.Alignment(x, y)` (π.χ. `-1, -1` για top-left).
